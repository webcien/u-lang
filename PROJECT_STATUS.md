## ğŸ”§ Technical Details

### Compiler Architecture
The U-Lang compiler follows a modular pipeline:

1. **Lexer** (`lexer.rs`): Tokenizes source code
   - Supports keywords: `fn`, `actor`, `let`, `var`, `return`, `await`
   - Handles literals: identifiers, strings, integers
   - Recognizes symbols: `()`, `{}`, `,`, `;`, `.`, `->`, `:`, `=`, `<`, `>`

2. **Parser** (`parser.rs`): Builds Abstract Syntax Tree (AST)
   - Implements EBNF grammar for U-Lang
   - Supports function declarations and actor declarations
   - Handles expressions: literals, identifiers, function calls, method calls

3. **Type Checker** (`type_checker.rs`): Enforces type safety
   - Implements ownership rules (single owner model)
   - Detects moved values and invalid ownership transfers
   - Maintains lexical scopes for variable resolution

4. **Code Generator** (`codegen/c.rs`): Generates portable C code
   - Translates U-Lang AST to C99-compatible code
   - Handles type mapping (i32 â†’ int, str â†’ const char*, etc.)
   - Special handling for `print()` â†’ `printf()`

5. **Linker**: Uses Zig's `cc` wrapper for cross-compilation
   - Target: `x86_64-linux-musl` (statically linked Linux binaries)
   - Supports: Linux, Windows, macOS, WebAssembly (WASI)
